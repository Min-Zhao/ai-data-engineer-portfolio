# Job Posting Alignment

This portfolio is aimed at the AI Data Engineer - LLMs / Agentic Systems profile.

## Required Signals

| Posting Requirement | Portfolio Evidence |
| --- | --- |
| Python programming | Three FastAPI backends with typed domain models and tests |
| TypeScript programming | Three React + TypeScript Vite frontends |
| Production-quality software | Health endpoints, tests, CI, Docker Compose, scoped services |
| FastAPI and React | Every app uses FastAPI and React |
| RAG systems | Pharma Research Agent Studio retrieves and cites scientific evidence |
| Conversational / agentic interfaces | Agent trace separates retrieval, synthesis, and review |
| Vector search / semantic retrieval | Local deterministic retrieval is implemented and documented for vector DB upgrade |
| Scientific workflows | Apps cover drug discovery, literature evidence, assays, ADME, biomarkers |
| Cross-functional collaboration | UX is framed for medicinal chemists, biomedical scientists, and research operators |
| GitHub portfolio required | Repo is organized as a portfolio with app matrix, CI, and clear run instructions |

## Interview Talking Points

- Explain why retrieval and synthesis are separated to improve traceability.
- Discuss how scientist-labeled evaluation sets would measure top-k retrieval quality.
- Describe migration from local retrieval to pgvector/OpenSearch/Qdrant.
- Explain how access control and audit logging would be required for internal research data.
- Show how frontend views make evidence review faster for non-engineering scientists.
